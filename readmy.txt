Для удобства работы, я сделал класс с базовыми командами бота в initial.py, а в каждом уроке (lsnx) я буду расширять
этот класс приделывая к нему изучаемый в этом уроке функционал.
==================================================================================================================
В боте есть три вида сигналов:
1)Commands, то есть команды вида "/start", обработка ведется по ним, как минимум
один такой обработчик start всегда есть в боте.
2)Текстовый фильтр, например "btn1", подходит для Reply кнопок (уроки lsn2, lsn3) и обычного текста введенного
пользователем.
3)Callback дата, это inline кнопки (которые высланы в чате см. lsn4), за этими кнопками также закреплен текст, который
извлекается по фильтру callback.

==================================================================================================================
1. ПРОСТЫЕ ТЕКСТОВЫЕ ОБРАБОТЧИКИ:
    lsn1 - создание пользовательских обработчиков комманд таких как "/start"

2.REPLY КНОПКИ (ПОД КЛАВИАТУРОЙ):

    lsn2 - reply кнопки обработка кнопок меню с помощью текстовых обработчиков. Цепочка кнопки: создание экземпляра кнопки
    с названием, установка кнопки на клавиатуре, отправка клавиатуры (сета кнопок) пользователю, объявление обработчика
    кнопки (text heandler).

    lsn3 - reply кнопки создание кнопок через build, более удобный способ создания кнопок и способ их выравнивания, сокрытие
    клавиатуры с кнопками после нажатия на любую из кнопок.


3.INLINE КНОПКИ (В ЧАТЕ):

    lsn4 - inline кноки кнопки которые появляются в чате и при нажатии на них выполняется действие без внешней отправки
    текста. Отправляется сигнал callback дата, который и вызывает действие.

    lsn5 - Пример toggle button, с помощью inline клавиатуры интерактивно на месте изменяем статус (меняем текст на кнопке,
    чтобы было видно что переключение произошло, например ✔ и ❌)

    lsn6 - Пример группы toggle button, создание множественного сета переключателей.
